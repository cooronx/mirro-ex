# 沪市逐笔重建为深市风格

目标不是简单改字段名，而是按“深市逐笔委托/逐笔成交”的理解方式，重建一份可直接使用的数据。

参考：

- 雪球《浅析上交所逐笔数据重建的方法》：https://xueqiu.com/1985440678/304033783

## 核心差异

根据上面的文章，沪市之所以要转成深市风格，关键是这两点：

1. 沪市逐笔委托里的 `volume` 不是原始申报量，而是“委托报入后、立即撮合剩余的量”。
2. 沪市撤单在委托表里，深市撤单通常体现在成交表里，成交类型为撤单。

所以这里真正要做的不是“字段改名”，而是“数据重建”。

## 重建原则

### 1. 委托表要还原成原始申报量

深市风格下，委托表里的数量应该表示这笔单子最初报进来的量。

沪市需要这样还原：

- 如果一笔委托报入后没有立即成交：
  - 委托量 = 沪市委托表原始 `volume`
- 如果一笔委托报入后部分立即成交：
  - 委托量 = 立即成交量 + 沪市委托表返回的剩余量
- 如果一笔委托报入后全部立即成交：
  - 沪市委托表可能没有这笔委托记录
  - 需要根据成交表补出一条“虚拟委托记录”
  - 委托量 = 该次立即成交量

### 2. 撤单要从委托侧搬到成交侧

为了变成深市风格，建议：

- 沪市委托表里的撤单记录，不再作为最终委托表输出
- 统一转成成交表中的一条撤单记录
- 这条撤单记录满足：
  - `deal_price = 0`
  - `deal_volume = 撤单量`
  - `deal_type = '4'`
  - `buy_syh` / `sell_syh` 至少一边为 `0`

### 3. 保留同一 channel 内的时序

重建后的数据仍然要保证：

- 同一个 `channel` 内顺序不乱
- 委托和成交共用同一套时序编号
- 如果补出了“虚拟委托记录”，它的顺序要落在对应成交之前

## 建议的目标表

### 逐笔委托表

```sql
create table L2_order
(
    channel              Int64,
    biz_index            Int64,
    code                 String,
    price                Decimal(20, 4),
    volume               Decimal(20, 4),
    direction            Int8,
    order_type           String,
    time                 DateTime64(3),
    order_number         Int64,
    EventDate            Date materialized toDate(time)
)
engine = MergeTree
partition by EventDate
order by (EventDate, channel, biz_index);
```

说明：

- `biz_index` 表示重建后的统一时序号
- `volume` 必须是“原始申报量”，不是沪市原始表里的剩余量
- `order_number` 保留原始委托号，后面成交关联还要用

### 逐笔成交表

```sql
create table L2_transaction
(
    channel_id         Int64,
    transaction_number Int64,
    code               String,
    deal_time          DateTime64(3),
    deal_price         Decimal(20, 4),
    deal_volume        Decimal(20, 4),
    buy_syh            Int64,
    sell_syh           Int64,
    deal_type          String,
    EventDate          Date materialized toDate(deal_time)
)
engine = MergeTree
partition by EventDate
order by (EventDate, channel_id, transaction_number);
```

说明：

- 正常成交：`deal_type = 'F'`
- 撤单成交：`deal_type = '4'`

## 映射规则

### 沪市委托 -> 深市风格委托

| 沪市字段 | 目标字段 | 规则 |
| --- | --- | --- |
| `channel` | `channel` | 直接映射 |
| `message_number` | `biz_index` | 直接映射，或重排后的统一序号 |
| `code` | `code` | 直接映射 |
| `price` | `price` | 直接映射 |
| `volume` | `volume` | 不能直接用，需要先重建成原始申报量 |
| `bs_flag` | `direction` | 买卖方向映射 |
| `order_type` | `order_type` | 类型映射 |
| `time` | `time` | 直接映射 |
| `order_number` | `order_number` | 直接映射 |

### 沪市撤单 -> 深市风格成交

| 沪市字段 | 目标字段 | 规则 |
| --- | --- | --- |
| `channel` | `channel_id` | 直接映射 |
| `message_number` | `transaction_number` | 直接映射，或重排后的统一序号 |
| `code` | `code` | 直接映射 |
| `time` | `deal_time` | 直接映射 |
| `0` | `deal_price` | 固定为 0 |
| `volume` | `deal_volume` | 撤单量 |
| `order_number` | `buy_syh / sell_syh` | 按买卖方向落一边，另一边置 0 |
| 常量 | `deal_type` | 固定为 `'4'` |

## 最简操作建议

如果你现在只是先把规则写进手册，先写下面三句就够了：

1. 沪市逐笔委托量要重建为原始申报量，不能直接拿剩余量当深市委托量。
2. 沪市完全立即成交但没有委托回报的单子，需要补委托记录。
3. 沪市撤单要从委托表挪到成交表，生成深市风格的撤单成交记录。

## 示意例子

下面三个例子只是用于说明重建规则，不是真实数据库样本。

### 例子 1：报单后部分立即成交

假设一笔买单原始报单 1000 股，价格 10.00。  
进入撮合后，先立即成交 300 股，剩下 700 股挂到订单簿。

沪市原始数据：

委托表

```text
channel=1
message_number=100
code=600000
price=10.00
volume=700
bs_flag=2
order_type=0
time=09:30:00.100
order_number=5001
```

成交表

```text
channel_id=1
transaction_number=99
code=600000
deal_time=09:30:00.100
deal_price=10.00
deal_volume=300
buy_syh=5001
sell_syh=4001
deal_type=F
```

重建后的深市风格数据：

委托表

```text
channel=1
biz_index=100
code=600000
price=10.00
volume=1000
direction=1
order_type=限价
time=09:30:00.100
order_number=5001
```

成交表不变

```text
channel_id=1
transaction_number=99
code=600000
deal_time=09:30:00.100
deal_price=10.00
deal_volume=300
buy_syh=5001
sell_syh=4001
deal_type=F
```

关键点：

- 沪市委托里的 `volume=700` 是剩余量
- 重建后委托量应为 `300 + 700 = 1000`

### 例子 2：报单后立即全部成交

假设一笔卖单原始报单 800 股，价格 9.98。  
订单一进来就全部成交，所以沪市委托表可能没有这条记录，只有成交表记录。

沪市原始数据：

委托表

```text
无
```

成交表

```text
channel_id=2
transaction_number=210
code=600000
deal_time=09:30:01.200
deal_price=9.98
deal_volume=800
buy_syh=6100
sell_syh=5200
deal_type=F
```

重建后的深市风格数据：

补一条委托表记录

```text
channel=2
biz_index=210
code=600000
price=9.98
volume=800
direction=2
order_type=限价
time=09:30:01.200
order_number=5200
```

成交表保留原记录

```text
channel_id=2
transaction_number=210
code=600000
deal_time=09:30:01.200
deal_price=9.98
deal_volume=800
buy_syh=6100
sell_syh=5200
deal_type=F
```

关键点：

- 沪市没有委托回报，不代表没有委托
- 为了变成深市风格，必须补出这条委托

### 例子 3：沪市撤单改造成深市撤单成交

假设一笔买单原来挂了 1000 股，后来撤掉 400 股。

沪市原始数据：

撤单记录出现在委托表

```text
channel=3
message_number=330
code=600000
price=10.01
volume=400
bs_flag=2
order_type=1
time=09:35:10.500
order_number=7001
```

重建后的深市风格数据：

这条记录不再放进最终委托表，而是转成成交表中的撤单记录

```text
channel_id=3
transaction_number=330
code=600000
deal_time=09:35:10.500
deal_price=0
deal_volume=400
buy_syh=7001
sell_syh=0
deal_type=4
```

如果是卖单撤单，则反过来：

- `buy_syh=0`
- `sell_syh=原 order_number`

关键点：

- 沪市撤单原本在委托表
- 重建后要挪到成交表
- `deal_price=0`
- `deal_type=4`
